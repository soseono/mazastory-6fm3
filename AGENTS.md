# MAZA BLOG — 에이전트 개발 가이드 (AGENTS.md)

> **이 문서는 마자 블로그 플랫폼의 핵심 철학, 비즈니스 모델, 기술 방향을 정의합니다.**
> 모든 AI 에이전트와 개발자는 이 원칙을 최우선으로 따릅니다.

---

## 1. 핵심 철학

> "구글과 싸우지 말고, 구글이 원하는 것을 줘라."

구글이 원하는 것 = **진짜 사람에게 진짜 도움이 되는 콘텐츠**

이 원칙만 지키면 구글이 트래픽을 무료로 보내준다. 마자 블로그의 모든 기능은
이 철학 위에 설계되어야 한다.

### 포지션 정의

> "유저가 콘텐츠의 **편집장**이 되고, 마자가 **편집팀 AI 직원** 역할을 한다."

- **구글 입장**: 사람이 운영하는 정상적인 블로그
- **애드센스 입장**: 규정 위반 없음
- **유저 입장**: 노력 대비 최고 수익
- **마자 입장**: 경쟁자가 복사할 수 없는 차별화된 SaaS

---

## 2. 비즈니스 모델: "Niche Authority Blog Network"

### 서브도메인 구조

```
유저가 aaa.com 을 소유
 ├── travel.aaa.com   ← 여행 특화 (한 주제만, 깊이 있게)
 ├── finance.aaa.com  ← 재테크 특화
 └── recipe.aaa.com   ← 요리 특화

각 서브도메인 = 독립된 전문 블로그 (Niche Site)
```

**핵심 SEO 전략: Topical Authority**
- 구글은 "한 주제에 집중된 전문 사이트"를 상위 노출 우대
- 같은 루트 도메인의 파워(백링크, DA)를 서브도메인이 공유 → SEO 레버리지

---

## 3. 콘텐츠 생산 파이프라인 (가장 중요)

```
마자스튜디오 AI → 초안 생성
                      ↓
              [필수] 유저가 3가지를 추가:
                  1. 본인 경험담 1문단
                  2. 실제 사진 1장 이상
                  3. 한국 현지 정보 (가격/주소 등)
                      ↓
              유저가 편집 완료 후 "발행 승인" 클릭
                      ↓
              GitHub Actions 트리거 → Astro Build
                      ↓
              Cloudflare Pages 자동 배포
                      ↓
              서브도메인 블로그에 글 반영
```

> **절대 원칙**: 사람의 "승인" 없이 자동 발행하지 않는다.
> 완전 무인 자동 발행은 구글 스팸 정책 위반으로 처리된다.

---

## 4. 발행 속도 가이드라인

| 방식 | 결과 |
|---|---|
| ❌ 하루 10개+ (완전 자동) | 구글 스팸 감지, 색인 제외 |
| ✅ 하루 1~2개 per 서브도메인 | 안전, 장기 성장 |
| ✅ 주 5개 정도 | 장기적으로 최적의 성과 |

---

## 5. 수익 구조 다각화 (애드센스만 의존 금지)

| 수익원 | 난이도 | 수익성 | 마자 자동화 지원 |
|---|---|---|---|
| 구글 애드센스 | 쉬움 | 낮음 | ✅ 자동 코드 삽입 |
| 쿠팡파트너스 | 쉬움 | 중간 | ✅ 상품 링크 자동 삽입 |
| 네이버 파트너스퀘어 | 중간 | 중간 | 🔜 예정 |
| 협찬/리뷰 포스팅 | 어려움 | 높음 | ❌ 유저 직접 |
| 정보성 PDF/전자책 판매 | 중간 | 높음 | 🔜 예정 |

> **마자 블로그 킬러 기능**: 수익 링크를 글 작성 시 자동으로 삽입해 주는 기능

---

## 6. 서브도메인 확장 로드맵 (유저에게 안내)

```
3개월: 서브도메인 1개 집중 (목표: 글 100개)
       → 애드센스 승인 조건 완성
6개월: 2~3개 서브도메인으로 확장
1년:   5~10개 운영 → 월 수익 안정화
```

## 7. 마자 블로그 플랫폼 핵심 기능 (구현 목표)

```
유저에게 제공해야 할 기능:
┌─────────────────────────────────────────────┐
│  1. 원클릭 Astro 블로그 서브도메인 생성     │
│  2. AI 초안 → 편집 UI → 예약 발행          │
│  3. 자동 sitemap.xml / robots.txt 생성      │
│  4. 수익 링크 자동 삽입 (애드센스, 쿠팡 등) │
│  5. 구글 서치콘솔 자동 연결 및 색인 요청   │
│  6. 통합 성과 대시보드 (방문자/수익 현황)  │
│  7. 서브도메인별 독립 분석 리포트          │
└─────────────────────────────────────────────┘
```

## 8. 기술 스택 원칙

- **프레임워크**: Astro (SSG) — 구글이 가장 좋아하는 정적 사이트 방식
- **배포**: Cloudflare Pages (무료, 빠름, 전 세계 CDN)
- **DNS**: Cloudflare (서브도메인 관리 자동화 가능)
- **빌드 트리거**: GitHub Actions (유저 승인 시 자동 빌드)
- **이미지**: Cloudflare R2 (외부 이미지 서빙)

---

## 9. 개발 에이전트 행동 원칙

1. **구글 정책 준수가 최우선이다.** 기능 개발 시 항상 `RULES.md`를 함께 확인한다.
2. **유저의 편집 승인 단계를 절대 제거하지 않는다.**
3. **콘텐츠 품질 보다 발행 속도를 우선하는 기능은 만들지 않는다.**
4. **각 서브도메인은 명확히 다른 주제와 독립된 sitemap을 가져야 한다.**
5. **수익 기능은 투명하게 공시한다** (광고 표기, 파트너 링크 표기 등).
